What is SocialFlow?
SocialFlow is a 100% self-hosted social media management platform that allows you to manage, schedule, and publish content across multiple social media platforms from a single dashboard. It supports Facebook, Instagram, YouTube, TikTok, Twitter, LinkedIn, Telegram, and Pinterest.
Is SocialFlow free?
Yes! SocialFlow is currently free to use. You only pay for your own infrastructure (server hosting). There are no monthly SaaS fees, no per-user charges, and no hidden costs.
What does "self-hosted" mean?
Self-hosted means you deploy and run SocialFlow on your own server infrastructure. Your data stays completely under your control - we never have access to it. This gives you complete privacy, security, and freedom.

üîß Setup & Installation

What are the system requirements?
Minimum Requirements:
  • Ubuntu 22.04/24.04 or similar Linux distribution
  • 2 GB RAM (4 GB recommended)
  • 20 GB disk space (more for media storage)
  • Node.js 20.x or higher
  • PostgreSQL 16
  • Nginx (for reverse proxy)
  • SSL certificate (Let's Encrypt recommended)
How do I install SocialFlow?
Installation involves several steps:
  1. Set up a server with Ubuntu 22.04+
  2. Install Node.js, PostgreSQL, and Nginx
  3. Clone the repository
  4. Configure environment variables
  5. Set up the database
  6. Build and deploy the application
  7. Configure OAuth for social platforms

Detailed instructions are available in the documentation at your deployment.

Do I need technical knowledge to set up SocialFlow?
Basic server administration knowledge is recommended. You should be comfortable with:
  • Linux command line
  • Basic database operations
  • Web server configuration
  • Environment variables

If you're not technical, consider hiring a developer for initial setup.

What is the Publishing Profiles feature?
Publishing Profiles allow you to save combinations of social media channels as reusable profiles. For example, if you manage multiple brands, you can create a "Brand A Profile" that includes all Brand A's channels across Facebook, YouTube, TikTok, etc. Then publish to all those channels with a single click.
Does SocialFlow support AI features?
Yes! SocialFlow integrates with OpenAI to provide:
  • Automated caption generation
  • Smart hashtag suggestions
  • Content optimization for different platforms

You need to configure your OpenAI API key to use these features.

Can I schedule posts?
Yes! You can schedule posts for future publishing. The scheduler runs automatically and publishes posts at the specified times. It includes retry mechanisms for failed posts.
What file types are supported for uploads?

üîê Account & Security

How is my data protected?
Your data is protected through:
  • HTTPS/TLS encryption for all connections
  • Encrypted password storage (bcrypt)
  • JWT token-based authentication
  • Database access controls
  • Regular security updates
  • Self-hosted infrastructure (you control access)
Are my social media access tokens secure?
Yes. Access tokens are:
  • Encrypted in the database
  • Never exposed in API responses
  • Only used for operations you request
  • Can be revoked at any time
  • Stored only on YOUR server (self-hosted)

üîß Troubleshooting

My post failed to publish. What should I do?
Common causes and solutions:
  • Token expired: Reconnect your social media account
  • Platform limits: Check if you exceeded posting limits
  • Content restrictions: Ensure content complies with platform policies
  • Media format: Verify file format and size are supported
  • Network issues: Check server connectivity

Check the Activity Logs for detailed error messages.

Why can't I connect my Instagram account?
Instagram requires a Business or Creator account linked to a Facebook Page. Steps:
  1. Convert your Instagram to Business/Creator account
  2. Link it to a Facebook Page
  3. Connect Facebook Page in SocialFlow
  4. Instagram will appear automatically
The scheduler is not running. How do I fix it?
Check the following:
  • Ensure the backend is running (pm2 list)
  • Check PM2 logs for errors (pm2 logs)
  • Verify database connection
  • Restart the backend (pm2 restart social-symphony-api)
I'm getting a "500 Internal Server Error". What does this mean?
This indicates a server-side error. Common causes:
  • Database connection issues
  • Missing environment variables
  • Permission errors
  • Backend crash or misconfiguration

Check the PM2 logs: pm2 logs social-symphony-api
